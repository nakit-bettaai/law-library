# พระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล (PDPA)
# Personal Data Protection Act (PDPA)

## ข้อมูลส่วนบุคคลคืออะไร / What is Personal Data?
ข้อมูลใดๆ ที่ทำให้สามารถระบุตัวบุคคลได้ ไม่ว่าทางตรงหรือทางอ้อม / Any information that can identify a person directly or indirectly.

ตัวอย่าง / Examples:
- ชื่อ-นามสกุล, ที่อยู่, เบอร์โทรศัพท์, อีเมล / Name, address, phone, email
- เลขบัตรประชาชน, หนังสือเดินทาง / National ID, passport number
- IP address, Cookie ID, ตำแหน่งที่ตั้ง / IP address, cookies, location data
- ข้อมูลการเงิน, ประวัติการซื้อ / Financial info, purchase history

## ข้อมูลอ่อนไหว / Sensitive Personal Data (Special Categories)
ต้องได้รับความยินยอมอย่างชัดแจ้ง / Requires explicit consent:
- เชื้อชาติ ชาติพันธุ์ / Race, ethnicity
- ความคิดเห็นทางการเมือง ศาสนา / Political opinions, religion
- ข้อมูลสุขภาพ / Health data
- พฤติกรรมทางเพศ / Sexual behavior
- ข้อมูลพันธุกรรม ชีวมิติ / Genetic, biometric data
- ประวัติอาชญากรรม / Criminal records

---

## สิทธิของเจ้าของข้อมูล / Data Subject Rights
1. **สิทธิได้รับแจ้ง / Right to be informed** — ทราบว่าเก็บข้อมูลอะไร ใช้ทำอะไร / Know what data is collected and how it's used
2. **สิทธิเข้าถึง / Right of access** — ขอดูข้อมูลของตนเอง / Request access to own data
3. **สิทธิแก้ไข / Right to rectification** — ขอแก้ไขข้อมูลที่ไม่ถูกต้อง / Request correction of inaccurate data
4. **สิทธิลบ / Right to erasure** — ขอให้ลบข้อมูล / Request deletion of data
5. **สิทธิคัดค้าน / Right to object** — คัดค้านการประมวลผล / Object to processing
6. **สิทธิระงับ / Right to restriction** — ขอระงับการใช้ข้อมูลชั่วคราว / Request temporary suspension
7. **สิทธิโอนย้าย / Right to data portability** — ขอรับข้อมูลในรูปแบบที่ใช้งานได้ / Request data in machine-readable format
8. **สิทธิถอนความยินยอม / Right to withdraw consent** — ถอนได้ตลอดเวลา / Can withdraw at any time

---

## หน้าที่ของผู้ควบคุมข้อมูล / Controller Obligations
- จัดทำนโยบายความเป็นส่วนตัว (Privacy Policy) / Maintain a Privacy Policy
- เก็บข้อมูลเท่าที่จำเป็น (Data Minimization) / Collect only necessary data
- รักษาความปลอดภัยของข้อมูล / Maintain data security
- แจ้งเหตุละเมิดภายใน 72 ชั่วโมง / Report data breach within 72 hours
- จัดทำบันทึกกิจกรรมการประมวลผล (ROPA) / Maintain Records of Processing Activities (ROPA)

---

## ฐานทางกฎหมายในการประมวลผล / Legal Bases for Processing
1. **ความยินยอม / Consent** — ต้องชัดแจ้ง เฉพาะเจาะจง / Must be explicit and specific
2. **การปฏิบัติตามสัญญา / Contract** — จำเป็นต้องใช้ข้อมูลเพื่อปฏิบัติตามสัญญา / Necessary to perform a contract
3. **หน้าที่ตามกฎหมาย / Legal obligation** — การปฏิบัติตามกฎหมาย / Compliance with legal duty
4. **ประโยชน์สาธารณะ / Public interest** — ภารกิจสาธารณะ / Public task
5. **ประโยชน์อันชอบด้วยกฎหมาย / Legitimate interests** — ไม่ละเมิดสิทธิเจ้าของข้อมูล / Must not override data subject's rights

---

## บทลงโทษ / Penalties
- **โทษทางแพ่ง / Civil**: ชดใช้ความเสียหายจริง + ไม่เกิน 2 เท่า (กรณีเจตนา) / Actual damages + up to 2x (if intentional)
- **โทษทางปกครอง / Administrative**: ปรับไม่เกิน 5 ล้านบาท / Fine up to 5 million THB
- **โทษทางอาญา / Criminal**: จำคุกไม่เกิน 1 ปี ปรับไม่เกิน 1 ล้านบาท / Up to 1 year imprisonment, 1 million THB fine (for sensitive data violations)
